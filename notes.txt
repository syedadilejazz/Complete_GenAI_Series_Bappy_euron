What is LLMs?
Large Language Models (LLMs) are foundational machine learning models that use deep learning algorithms to process and understand natural language.
These models are trained on massive amount of text data to learn patterns and entity relationship in the language.
It is a language model which is responsible for performing tasks such as:
-text to text generation
-text to image generation
-image to text generation (Multimodal)

What makes LLM so powerful?
In case of LLM, one model can be used for a whole variety of tasks like:
-Text generation
-Chatbot
-Summarizer
-Translation
-Code generation and more

So, LLM is subset of Deep Learning & it has some properties merge with Generative AI.
Train the model for a specific task.

-GPT(OpenAI)
-xLM(Meta)(Designed for multilingual understanding)
-T5(Google)(Text-To-Text Transfer Transformer)
-Llama(Meta)(Developers could:download,fine-tune,run locally)
-Mistral(Mistral AI)(Smaller models but very strong performance.)
-Falcon(Technology Innovation Institute)

This is one of the most important concepts in GenAI.
People often confuse:
-Open-source
-Open-weight
-Closed-source
-Research models

They are NOT the same thing.
1. Closed Models
A closed model means:
You cannot download the actual model weights
You cannot see full training data
You cannot fully inspect training code
You mainly access it through an API or app

Example:
OpenAI GPT
Google Gemini
Anthropic Claude

Think of it like this
You can USE the AIBut you cannot OPEN the AI engine
Like:
You can drive a car
But cannot open factory design documents
What companies provide
Usually:
-Chat interface
-API access
-Some documentation
But NOT:
-Full model internals
-Weights
-Full training dataset
Advantages
-Very powerful
-Professionally maintained
-Usually best performance
-Safer and optimized
Disadvantages
-Expensive API cost
-No deep customization
-Internet required
-Company controls usage

2. Open-Weight Models
This is VERY important.
Open-weight does NOT always mean fully open-source.
It means:
The trained neural network weights are downloadable.
Example:
Meta Llama
Mistral AI Mistral
Technology Innovation Institute Falcon

What are “weights”?
Weights are the learned knowledge of the AI.
During training:
model learns patterns,numbers get adjusted,these learned numbers = weights

Simplified idea:
Training Data→Learning Process→Model Weights\text{Training Data} \rightarrow \text{Learning Process} \rightarrow \text{Model Weights}Training Data→Learning Process→Model Weights
Weights store:
language understanding,reasoning patterns,coding knowledge and relationships between words

Open-weight means you can:
-Download model
-Run locally
-Fine-tune it

Modify behavior
-Deploy on your own servers

But maybe NOT:
-Get training dataset
-Get full training code

Redistribute commercially freely
So:
Open-weight ≠ Fully open-source

3. Fully Open-Source Models
True open-source means:
-weights available
-training code available
-architecture available
-dataset details available
-modification allowed


This is similar to Linux/open-source software philosophy.

Example spirit
Like:
-Linux
-Python
-TensorFlow
You can:
-inspect
-modify
-redistribute

In AI, fully open-source is rare
Because training huge models costs:
millions of dollars
massive GPU clusters
proprietary datasets

So companies often release:
weights only
instead of everything.

4. Research/Open Models
This means:

Mainly released for academic or research purposes.
Example:
T5
XLM

Usually includes
Research paper
Model architecture
Some pretrained weights
Benchmarks
Experimental ideas

Purpose:
Advance AI researchNot necessarily commercial productization

Why T5 became famous
T5 introduced:
Everything is Text-to-Text

So its research impact became huge.

End to End Generative AI Pipeline
Generative AI pipeline is a set of steps followed by build an end to end GenAI Software.
Break the problem down into several sub-problem, then try to develop a step by step procedure to solve them. Since, language processing is invoolved , we would also list all the forms of text processing needed at each steps. This step by step processing of text is known as pipeline.
-Data Acquisition
-Data Preparation
-Feature Engineering(Text Representation)
-Modelling
-Evaluation
-Deployment
-Monitoring & Model Updating
